PART 4: EXECUTION FLOW
================================================================================

When user runs: python main.py --case-data ./case_data --verbose

1. Print banner with rich formatting
2. Parse all command-line arguments
3. Validate environment (Python, tools, packages, directories)
4. Create session ID (uuid4)
5. Initialize StructuredLogger (auto-creates logs directory)
6. Start MCP server in background (asyncio task)
7. Wait 1 second for server initialization
8. Initialize TriageAgent with MCP URL
9. Run 7-phase triage:
   - RECONNAISSANCE
   - DISK_ANALYSIS
   - MEMORY_ANALYSIS
   - LOG_ANALYSIS
   - CORRELATION
   - SELF_CORRECTION
   - REPORT
10. Score findings against ground truth (if provided)
11. Generate markdown report with 9 sections
12. Save both markdown and JSON reports
13. Save full execution trace (JSON + markdown)
14. Print rich summary table with metrics
15. Cleanup: cancel server task, suppress CancelledError
16. Exit with code 0

Output files created:
- logs/session_{id}.json (active session log)
- logs/session_{id}_full_trace.json (complete trace)
- logs/session_{id}_summary.md (readable summary)
- reports/accuracy_report_{timestamp}.md (markdown)
- reports/accuracy_report_{timestamp}.json (machine-readable)
